# Workflows

These guides show how command families fit together.

Use workflows when you know the outcome you want but not the exact command
family. Each guide starts from read or inspection commands, then points out
where state changes happen and when automation should use `--json`, `--dry-run`,
or `--yes`.

- [First Run](./first-run.md)
- [Multi-provider Profiles](./multi-provider-profiles.md)
- [Repository Lifecycle](./repository-lifecycle.md)
- [Change Review](./change-review.md)
- [Pipeline Delivery](./pipeline-delivery.md)
- [Security Governance](./security-governance.md)
- [Workspace Fleets](./workspace-fleets.md)
- [Automation and JSON](./automation-json.md)

For exact flags, pair these guides with `gitfleet help [command...]` and the
[Command Reference](../commands/README.md).
